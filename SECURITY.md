# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: **diggerwp@gmail.com**

Please include the following information:

- Type of vulnerability (e.g., SQL injection, XSS, CSRF)
- Location of the affected source code (file and line if possible)
- Steps to reproduce the vulnerability
- Proof-of-concept or exploit code (if available)
- Impact assessment

## Response Timeline

You should receive an initial response within 48 hours. We will:

1. Confirm receipt of your report
2. Assess the severity and impact
3. Work on a fix
4. Release a patch and notify you
5. Credit you in the release notes (unless you prefer anonymity)

## Security Best Practices

When using this package:

- Keep dependencies updated
- Use the latest stable version
- Follow WordPress security guidelines
- Enable WordPress debug mode only in development

## Disclosure Policy

We follow responsible disclosure:

1. Security issues are fixed before public disclosure
2. Fixes are released as patch versions
3. CVE identifiers are requested for significant issues
4. Security advisories are published after fixes are available

Thank you for helping keep WPZylos secure!
